# Theme Editable Journal

根据照片主题自动匹配视觉风格，生成可继续编辑的电子手帐 / Plog。

这个 skill 适合把一组旅行、亲子出行、博物馆、展览、动物园、城市漫游等照片，整理成适合小红书、朋友圈长图或个人相册记录的可编辑 HTML 手帐。HTML 页面调整完成后，可以直接导出下载为 PNG 图片，实现所见即所得保存。

## 谁适合使用

- 带孩子逛博物馆、动物园、展览后，想把照片整理成有故事感手帐的家长
- 想做小红书 Plog、旅行路线图、观展记录、亲子研学记录的内容创作者
- 有一组照片，但不确定应该用什么风格、怎么排版、怎么写短文案的人
- 希望先由 AI 生成初稿，再自己微调照片位置、文字和模块的人

## 适用场景

- 自然博物馆、历史博物馆、书画馆、美术馆、科技馆
- 动物园、植物园、海洋馆
- 城市旅行、周末路线、亲子游、成人旅行
- 各类主题展览、文化空间、生活方式照片记录

## 这个 skill 的优点

- **自动匹配主题风格**：会根据照片内容选择自然科普、书画水墨、历史复古、动物园观察日记、城市旅行等不同视觉路线。
- **不是固定模板套图**：底纹、配色、文字气质、装饰元素都会跟照片主题呼应。
- **先出干净版本**：默认先生成不拥挤、不堆贴纸的 clean 版本，再按需要增加装饰。
- **支持用户继续编辑**：生成的是 HTML 编辑页，可以拖拽模块、旋转、缩放、裁剪、替换图片、编辑文字。
- **支持新增模块**：可以增加文字描述、图文模块、时间胶囊，并让中轴时间线自动延长。
- **HTML 可导出图片**：调整完成后点击下载按钮，即可把当前 HTML 手帐导出为 PNG 图片。
- **强调所见即所得导出**：导出的图片会保留当前调整后的页面状态，包括照片位置、裁剪范围、旋转角度、文字修改和新增模块。
- **内置审美约束**：重点规避文字重叠、图文重叠、主体裁掉、底纹不贴主题、贴纸堆砌等问题。

## 使用方法

### 1. 安装到 Codex skills 目录

把整个文件夹放到：

```bash
~/.codex/skills/theme-editable-journal
```

目录结构应类似：

```text
theme-editable-journal/
├── SKILL.md
├── agents/
├── assets/
├── references/
└── scripts/
```

### 2. 在 Codex 中调用

新开一个 Codex 线程，上传或提供照片文件夹，然后说：

```text
用 $theme-editable-journal，把这组动物园照片做成可编辑电子手帐。
```

也可以更具体：

```text
用 $theme-editable-journal，把这组书画馆照片做成适合小红书发布的电子手帐，风格要清冷、留白多、文字简短。
```

或：

```text
用 $theme-editable-journal，把这组旅行照片做成路线感 Plog，不一定以孩子为主体，整体更成人、克制一点。
```

### 3. 生成后怎么编辑

生成的 HTML 页面通常会支持：

- 点击文字直接改文案
- 点击照片后替换图片
- 拖拽图文模块调整位置
- 调整照片旋转、缩放、裁剪范围
- 新增文字描述模块
- 新增图文模块
- 删除选中的模块
- 新增时间点，并让中轴时间线适配延长
- 点击下载按钮导出当前调整后的 PNG

也就是说，最终不只是得到一个网页预览，而是可以把调整好的 HTML 手帐完整下载成图片，用于发布、保存或继续分享。

### 4. 照片素材建议

更容易生成好看的手帐素材包括：

- 一两张大场景照片：入口、展厅、路线、建筑、环境
- 几张人物观察照片：背影、侧脸、仰头看、指向展品
- 几张细节照片：展品、动物、纹理、票据、文字、手部动作
- 尽量避免严重模糊、主体太远、画面杂乱、广告牌和路人过多的照片

## 输出结果

通常会得到：

- 一个可本地打开或本地服务运行的 HTML 编辑页面
- 配套图片和脚本资源
- 可通过页面按钮导出的 PNG 长图，导出结果应与当前 HTML 页面保持一致

## 设计原则

这个 skill 会优先保证：

- 图和文不重叠
- 文和文不重叠
- 照片主体清楚
- 图文关系明确
- 主题底纹和照片内容呼应
- 文案短句、克制、有完整小故事感
- 用户可以继续调整，而不是只能接受一张静态成品图
